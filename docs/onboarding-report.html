03 — UI

Design system overhaul

The biggest visible change. The dashboard went from an ad-hoc collection of styles (Arial/Times, inline rules, emoji nav icons) to a tokenised design system applied across every page.

Δ 1

Design tokens

style.css: +708 lines
bar.css: +365 lines

A :root block now declares every recurring value as a custom property. Pages and components reference the tokens instead of hard-coding hex values or pixel offsets.

:root {
  --vc-brand-{900,700,500,100}      /* brand scale */
  --vc-accent-{500,400}             /* accent scale */
  --vc-ink-{900,700,500,300,100}    /* neutral ink */
  --vc-bg, --vc-surface, --vc-border
  --vc-success-{700,500,100}        /* status: success */
  --vc-warn-{700,500,100}           /* status: warn   */
  --vc-danger-{700,500,100}         /* status: danger */
  --vc-font-sans, --vc-font-mono
  --vc-fs-{xs,sm,md,lg,xl,2xl,3xl}  /* type scale  */
  --vc-s-{1,2,3,4,5,6,8}            /* 4 px spacing */
  --vc-r-{sm,md,lg}                 /* radius     */
  --vc-shadow-{sm,md,lg}            /* elevation  */
}

Δ 3

Page rewrites

index.py, home.py, runs.py, truvari.py

All four Dash pages were re-laid-out to consume the design system. Concrete changes:

  • Landing (/): tokenised hero band, recent-runs panel, three workspace action cards. Empty and error states render as designed alert blocks instead of raw traceback text.
  • /home: ships the shared site header so users can navigate back. Sidebar nav icons replaced with letter marks (Sum, SV, VC, CNV, ROH, HT, Plo, Bed, Cov, Map). The right rail collapsed shows a vertical MANUAL STATUS marker; expanded reveals the controls.
  • /runs: unified header, cleaner tabs, AWS S3 import section restyled with tokens, ZIP upload form rebuilt with a custom file drop zone and a 2 × 2 benchmarking option grid.
  • /truvari: shared header and design tokens for visual consistency.
Δ 4

Accessibility & resilience

Cross-cutting
  • Inter system font stack with text-wrap: balance on headings — replaces the previous Arial/Times default chain.
  • Every interactive element exposes a :focus-visible ring built from --vc-brand-500.
  • prefers-reduced-motion: reduce collapses every transition to ~0 ms.
  • Touch targets meet the 44 px minimum on the responsive breakpoint.
  • Status colors avoid red/green-only encoding; an amber warning tier is provided.
  • Dash callbacks now timeout after 4 s on reads and 10 s on POST, so a stalled API no longer freezes the page.
04 — Backend

Backend stabilization

A pair of merges (backend-stabilization #4, #5) introduced two new modules at the heart of the API and hardened uploads against malicious archives.

RBAC at the door

qc-dashboard/api/app/security.py (NEW, 90 lines)
  • Three roles: OPERATOR < REVIEWER < ADMIN.
  • API-key auth driven by env VCBENCH_API_KEYS (format role:key,role:key).
  • FastAPI dependency require_role(*allowed) wired into write endpoints (e.g. POST /qc_metrics, POST /upload/aws, POST /runs/{name}/benchmarking).
  • Local development remains usable with VCBENCH_AUTH_DISABLED=1 — shared deployments configure real keys.
  • Includes hash_password built on hashlib with secrets-sourced salt.

Centralised settings

qc-dashboard/api/app/settings.py (NEW, 46 lines)
  • Single import surface for filesystem paths (LAB_RUNS_DIR, PROCESSED_DIR, REFERENCE_DIR, TMP_DIR, UPLOAD_DIR).
  • Reads DATABASE_URL from env, with the production-correct default (port 55433).
  • Defines upload-hardening knobs: MAX_UPLOAD_BYTES (20 GB), MAX_ZIP_MEMBERS (5000), MAX_EXTRACTED_BYTES (100 GB).
  • All knobs are env-overridable through VCBENCH_* variables.

Upload hardening

qc-dashboard/api/tasks/upload_run.py (+180 lines)
  • Streaming uploads with explicit byte-count enforcement against MAX_UPLOAD_BYTES.
  • Zip extraction validated against MAX_ZIP_MEMBERS and MAX_EXTRACTED_BYTES — closes the zip-bomb vector.
  • Member paths sanitised so an archive cannot write outside its target directory (path-traversal protection).

Run lifecycle tracking

qc-dashboard/api/tasks/process_run.py (+82 lines)
  • Background processing now records processing_started_at / processing_completed_at on the run row.
  • Failures persist error_message instead of disappearing silently into logs.
  • Status transitions guarded against double-start.

06 — API

API surface clean-up

Two latent bugs in the FastAPI routers were resolved during the stabilization pass, alongside the introduction of env-driven configuration in place of hard-coded URLs.

Fix A

Double-prefixed paths

happy_metrics.py, qc_metrics.py

Two routers declared their routes with a hard-coded /api/v1/ prefix while also being mounted by main.py with prefix="/api/v1". The effective URLs were /api/v1/api/v1/runs/… — silently unreachable from outside.

- @router.post("/api/v1/runs/{run_name}/happy_metrics", …)
- @router.get("/api/v1/qc_metrics/{run_id}/{metric_name}", …)
+ @router.post("/runs/{run_name}/happy_metrics", …)
+ @router.get("/qc_metrics/{run_id}/{metric_name}", …)
Fix B

Dead route removed

runs.py

An old @router.get("/runs/{metric_name}") sat after /runs/{run_id} in the same file. FastAPI matches by declaration order, so any request to /runs/<anything> was shadowed by {run_id}. The handler also returned nothing — pure dead code, removed.

Fix C

Env-driven base URL

commit b23454d · dash_app/config.py

The Dash client used to hard-code http://localhost:8000/api/v1. The configuration now reads API_HOST and API_PORT from the environment, with sensible defaults, so the same code runs unchanged in dev, staging, and production.

- API_BASE_URL = "http://localhost:8000/api/v1"
+ API_HOST     = os.getenv("API_HOST", "127.0.0.1")
+ API_PORT     = os.getenv("API_PORT", os.getenv("PORT", "8002"))
+ API_BASE_URL = os.getenv("API_BASE_URL", f"http://{API_HOST}:{API_PORT}/api/v1")
Surface

Final route count

From OpenAPI spec

22 path/method pairs documented via OpenAPI, plus the /ws/download/{sample_id} WebSocket. Every previously documented endpoint is still present; the cleanup is purely making them reachable as advertised.

07 — Pipeline

Pipeline hardening

One pipeline script received a security and ergonomics refactor; the rest are byte-identical with the November state.

pipeline/vcf_filter.py — three improvements

  • No more shell=True. The previous code passed a constructed shell string to subprocess.run(shell=True) — a command-injection vector if a sample name ever contained metacharacters. Replaced with an argv list ["bcftools", "query", "-f", "%TYPE\\n", str(vcf_path)].
  • pathlib instead of os.path. Relative paths replaced with PROCESSED_DIR = Path(__file__).resolve().parents[1] / "data" / "processed", so the script behaves the same regardless of the caller's CWD.
  • Encapsulated in main(). Logic that ran at import time now sits inside a function with a guarded __main__, making the module importable and testable.
- output_path = os.path.join(PROCESSED_DIR, "variant_summary.tsv")
- cmd = f'bcftools query -f "%TYPE\\n" {vcf_path} | grep -w "{vtype}" | wc -l'
- result = subprocess.run(cmd, shell=True, capture_output=True, text=True)
+ output_path = PROCESSED_DIR / "variant_summary.tsv"
+ result = subprocess.run(
+     ["bcftools", "query", "-f", "%TYPE\\n", str(vcf_path)],
+     check=False, capture_output=True, text=True,
+ )
+ pattern = re.compile(rf"\b{re.escape(vtype)}\b")
+ return sum(1 for line in result.stdout.splitlines() if pattern.search(line))

10 — Verdict

Where this leaves the project

No feature was removed

Every REST endpoint, dashboard page, pipeline script, and CLI helper from November 2025 still exists in May 2026. Two route signatures changed only because they had been bugged from the start (double prefix); one route was dead code and was removed without behavioural impact.

Schema additions are backward-compatible

Four new columns on LabRun, plus cascade-delete foreign keys on the metric tables. Existing rows continue to read without migration of legacy data; new columns are nullable or have server-side defaults.

The codebase is meaningfully more secure

Three real attack surfaces narrowed: API now defends with role-based keys, uploads cap size/members/extraction (zip bombs and path traversal), and the pipeline drops shell=True. None of these existed in November.

The dashboard is meaningfully more polished

A tokenised design system, Inter typography, focus-visible rings, reduced-motion support, and a coherent set of UI primitives replace the previous ad-hoc styles. Pages share a navigation chrome and behave consistently across breakpoints.

Carried-over residue worth a follow-up

The README quick start still had a handful of defaults that didn't match the code (env name, ports, route table) at the end of the May 5th merges — those were the subject of the more recent onboarding pass and are out of scope for this report.
