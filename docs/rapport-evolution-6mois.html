API — corrections d'URL silencieuses

Deux routers déclaraient leurs paths avec un préfixe /api/v1/ alors qu'ils étaient déjà montés avec ce préfixe. Les URLs documentées renvoyaient 404.

Avant — 2025-11-18
# happy_metrics.py
@router.post("/api/v1/runs/{run_name}/happy_metrics")
@router.get("/api/v1/runs/{run_name}/happy_metrics")

# main.py
app.include_router(happy_metrics.router,
                   prefix="/api/v1")

# URL effective :
/api/v1/api/v1/runs/{run_name}/happy_metrics
                → 404 sur le path du README
Après
# happy_metrics.py
@router.post("/runs/{run_name}/happy_metrics")
@router.get("/runs/{run_name}/happy_metrics")

# main.py (inchangé)
app.include_router(happy_metrics.router,
                   prefix="/api/v1")

# URL effective :
/api/v1/runs/{run_name}/happy_metrics
                → 200 conforme au README

Bonus : code mort retiré

GET /runs/{metric_name} était déclarée après GET /runs/{run_id}. FastAPI matche par ordre de déclaration : la première gagnait toujours. La fonction ne retournait rien de toute façon.

Pipeline — refactor vcf_filter.py

Seul script du pipeline/ à avoir bougé. Refactor cosmétique et correctif sécurité.

Avant
import os
import subprocess

PROCESSED_DIR = "../data/processed/"

def count_variants_by_type(vcf_path, vtype):
    if not os.path.exists(vcf_path):
        return 0
    cmd = f'bcftools query -f "%TYPE\\n" {vcf_path} | grep -w "{vtype}" | wc -l'
    result = subprocess.run(cmd, shell=True,
                            capture_output=True, text=True)
    return int(result.stdout.strip())
Après
import csv, re, subprocess
from pathlib import Path

PROCESSED_DIR = Path(__file__).resolve().parents[1] / "data" / "processed"

def count_variants_by_type(vcf_path, vtype):
    if not vcf_path.exists():
        return 0
    result = subprocess.run(
        ["bcftools", "query", "-f", "%TYPE\\n", str(vcf_path)],
        shell=False, check=False, capture_output=True, text=True,
    )
    if result.returncode != 0:
        return 0
    pattern = re.compile(rf"\b{re.escape(vtype)}\b")
    return sum(1 for line in result.stdout.splitlines()
               if pattern.search(line))

Trois gains

  1. Sécurité : shell=True retiré → plus d'injection possible via un chemin malveillant.
  2. Portabilité : pathlib.Path, plus de chemin relatif fragile "../data/...".
  3. Robustesse : regex Python remplace grep | wc -l, gestion explicite de returncode et FileNotFoundError.

Tous les autres scripts du pipeline (happy.sh, truvari.sh, eval_avg.py, stats_avg.py, reformat_csv.py, summary_csv.py) ainsi que script/aws_download_gvcf.sh et script/setup_reference.sh sont identiques au byte près.

Frontend — refonte UI tokenisée

Toutes les pages Dash partagent désormais un système de design unifié déclaré dans dash_app/assets/style.css.

Tokens introduits sous :root

  • Brand : --vc-brand-{900,700,500,100}
  • Ink & neutres : --vc-ink-{900,700,500,300,100}
  • Status : success / warn / danger, 3 paliers chacun
  • Typo : Inter, échelle xs → 3xl
  • Spacing : 4 px scale (--vc-s-1..8)
  • Radius, shadow, layout

Primitives réutilisables

  • .btn + variants primary / secondary / success / ghost
  • .action-card — cartes de navigation (page /)
  • .runs-panel — table des runs récents
  • .section-card — surface générique
  • .pill.is-success / .is-warn / .is-danger
  • .alert, .empty-state, .error-state

Résilience des callbacks

Tous les appels HTTP du Dash vers l'API ont maintenant un timeout 4 s en lecture, 10 s en POST. Une API qui broute ne fige plus la page.

Accessibilité

  • Stack système Inter avec text-wrap: balance sur les titres.
  • Anneau :focus-visible systématique en --vc-brand-500.
  • Support prefers-reduced-motion → transitions à ~0 ms.
  • Cibles tactiles ≥ 44 px sur le breakpoint 900 px.
  • Niveau warn ambré pour éviter le red/green-only.
